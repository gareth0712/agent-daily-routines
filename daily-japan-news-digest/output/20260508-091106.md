### [Claude Code Auto Modeの設計判断 — AI分類器に何を委ね、何を委ねないか](https://zenn.dev/miyan/articles/claude-code-auto-mode-safeguard-design-2026)
*Claude Code Auto Mode の設計決定：AI 分類器に何を委任し、何を委任しないか*
來源：Zenn - Claude Code｜發布：2026-05-07 20:42 JST

Auto Mode は「デフォルト承認疲れ」と `bypassPermissions`（無制限）の中間に位置する第3の信頼モデルで、AI 分類器によって自律性と安全性のバランスを取る。2段階分類パイプラインを採用しており、Stage 1 では Sonnet 4.6 による高速フィルタリング、Stage 2 では疑わしいアクションへの Chain-of-Thought 推論が行われる。誤拒否率（FPR）は0.4%（500操作あたり約2件のブロック）、誤検知率（FNR）は過剰操作17%・データ流出試行5.7%と発表されているが、サンプルサイズ（n=52）が小さいため実際の性能幅は7〜30%に達する可能性がある。デフォルトの信頼境界はワークスペースディレクトリのみで、外部API・他リポジトリ・クラウドリソースは明示的な信頼付与なしにブロックされ、3回連続拒否または累計20回拒否でセッションが手動承認に戻る設計。⚠️ AI 分類器の非決定的な性質・Anthropic によるベンダーロックイン（基準が外部検証不可）・「善意の過剰操作」見逃しリスクがあり、本番インフラ変更・CI/CD ヘッドレスモードには不適とされる。推奨実装は OS サンドボックス（Linux bubblewrap / macOS seatbelt）＋ deny ルール＋ AI 分類器の3層防御で、`plan` → `acceptEdits` → `auto` の段階的展開が推奨される。
