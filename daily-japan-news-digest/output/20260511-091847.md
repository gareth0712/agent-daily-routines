### ⚠️ [Claude Code拡張 v2.1.136 が Windows で起動しなくなった話と、1分で確かめる方法](https://zenn.dev/yurukusa/articles/602052fec1ec68)
*VS Code 拡張 v2.1.136 の Windows 起動不可バグと確認・対処法*
来源：Zenn - Claude Code｜発布：2026-05-10 JST

⚠️ VS Code 拡張「Claude Code **v2.1.136**」が Windows 環境で起動不可になるバグが発生。原因はビルド時に Linux 開発環境のパス（`/home/runner/work/claude-cli-internal/...`）がバンドル内に定数として埋め込まれ、Windows がパス形式を認識できずに `TypeError` で拡張が無効化されるため。Linux・macOS ユーザーには影響なく、**Windows ユーザーのみ**が対象となる。VS Code の Developer Logs で「`/home/runner/work/`」の文字列を確認すれば 1 分で影響を判定できる。対処法は PowerShell でバンドルファイル内の不正パスを置き換えるか、**v2.1.135 へのダウングレード**が推奨される。
