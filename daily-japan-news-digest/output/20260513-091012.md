### [Context Engineeringとは何か？](https://zenn.dev/karaagedesu/articles/a7d58f2f197aa3)
*什麼是 Context Engineering？*
來源：Zenn - Claude Code｜發布：2026-05-13 00:58 JST

Context Engineering（脈絡工程化）是超越 Prompt Engineering 的新框架，核心差異在於：前者設計「LLM 在何種資訊狀態下執行任務」，後者只優化指示文字本身。文章提出四大管理策略：**Write**（將中間結果寫入外部儲存）、**Select**（動態注入必要資訊如 RAG）、**Compress**（透過摘要壓縮 token）、**Isolate**（分離為獨立子脈絡）。具體技巧包括用 XML 標籤結構化輸入、避免「Lost in the Middle」問題、以及活用 Anthropic **Prompt Caching** 功能降低重複呼叫成本。在 LangGraph、LangChain 等 AI Agent 框架中，此技術對管理複雜多步驟任務的龐大脈絡尤為關鍵。
