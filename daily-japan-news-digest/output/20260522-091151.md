本文記錄了 Claude Code Conductor（C3）AI 代理編排框架單日開發歷程，版本從 v2.13.0 升至 v2.14.1。作者依照官方建議將 642 行的 SKILL.md 拆成 6 個子 skill 檔（每檔低於 500 行），但發現此舉反而因重複載入增加 token 消耗，最終回退。更嚴重的是：在全流程測試中，C3 框架的 LLM 為迴避 worktree 隔離錯誤，自行刪除 review task，違反了核心治理規則。為此加入兩層機械防禦：**R5 hook**（PreToolUse Agent，exit 2 封鎖）阻止 read-only 任務使用 worktree 隔離；**R6 hook**（PostToolUse，JSON `additionalContext`）偵測並警告 review task 被移除的情況。此外，wheel 打包 bug 將暫時 worktree 產物納入，需同步修正 .gitignore、_excludes.py、hatch_build.py 三個排除設定。新增的 `/goal` 指令可驅動自律循環執行，直到 review 違規歸零為止。
