### [# Hooksで自動化する——「毎回同じ指示」から卒業する【中級Ch4】](https://zenn.dev/shun_producer/articles/1c63765ff7524a)
*用 Hooks 自動化——告別「每次重複相同指示」【中級 Ch.4】*
來源：Zenn - Claude Code｜發布：2026-06-01 11:37 JST

Hooks 是將特定事件與自動動作串聯的機制，在 `~/.claude/settings.json` 中定義觸發條件與對應的 shell 指令，設定後無需每次手動下指示。三種主要 Hook 類型：**PreToolUse**（工具使用前，適合權限檢查或日誌記錄）、**PostToolUse**（工具完成後，適合格式化或自動 git commit）、**Stop**（Claude 回應結束時，常用於完成通知）。文章示範三個實用案例：macOS 通知（`osascript`）、Edit/Write 工具完成後自動執行 `git commit`、Python 程式碼自動以 `black . --quiet` 格式化。建議新手從單一 Stop Hook 開始，避免在 Hook 中執行超過 3 秒的重型處理，設定變更後需重啟 Claude Code 生效。
