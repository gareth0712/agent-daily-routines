⚠️ 在 Claude Code 的 `SessionEnd` Hook 中呼叫 `claude` 會造成無限迴圈：每次新工作階段結束都會觸發相同 Hook，導致遞迴呼叫。作者確認每次觸發的 `session_id` 均不同，這是因為 Hook 本身啟動了新工作階段。實際影響：Hook 同步執行且會阻塞操作，數分鐘內觸發數百次，耗盡每 5 小時的使用配額，並使 `/clear` 命令凍結長達兩分鐘。防禦策略：避免在出口設置守衛（依賴成功執行留下的標記），改在入口檢查環境變數；使用 `async` 設定讓重型作業在背景執行；以 `exit code 2` 強制阻止危險操作；採用 `PreToolUse` Hook 搭配 deny 回傳進行預防性阻止。部署前必須驗證：觸發頻率、同步/非同步性質、遞迴可能性，以及暴走時的損害控制機制。
