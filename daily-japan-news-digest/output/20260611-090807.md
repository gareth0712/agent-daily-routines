### [Coding Agentのコマンド実行をBash ASTベースで検査するツール「Vetol」を作った](https://zenn.dev/tf63/articles/vetol-tf63-20260610)
*打造以 Bash AST 為基礎檢驗 Coding Agent 命令執行的工具「Vetol」*
來源：Zenn - Claude Code｜發布：2026-06-10 15:23 JST

**Vetol** 是一個透過解析 Bash AST（抽象語法樹）來驗證 Coding Agent 命令安全性的工具，能偵測隱藏在命令替換、`&&`、`||` 等巢狀結構中的危險指令，解決了傳統字串比對無法防範的繞過攻擊。工具支援兩種模式：**Allowlist 模式**（只有明確允許的命令才執行，其他全部封鎖）和 **Denylist 模式**（封鎖指定危險命令，其他放行）。規則設定可透過 JSON 組態檔指定 `include`/`exclude` 欄位，細粒度控制 `-f`、`--force` 等旗標的允許狀態。整合方式為掛接 Claude Code 的 **PreToolUse Hook**：Hook 接收命令輸入後呼叫 Vetol 進行驗證，根據 JSON 組態決定批准或拒絕並附上說明。⚠️ 限制：透過 `bash -c`、`eval`、`python -c` 等直譯器動態執行的命令無法被 AST 分析攔截，需另外在 blocklist 中處理。
