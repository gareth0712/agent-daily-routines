### [AIエージェントのコスト削減、いちばん危ないのは「モデルを落とす」だった](https://zenn.dev/hayua/articles/700aabeba45500)
*AI 代理成本優化：降級模型才是最危險的選擇*
來源：Zenn - Claude Code｜發布：2026-07-03 04:03 JST

作者指出，在 AI 代理成本優化策略中，「降級模型」（如從 Opus/Sonnet 換成 Haiku）看似直觀，實際上是最危險的選項。程式碼診斷與重構等決策型任務若使用輕量模型，修復迴圈（fix loop）反覆觸發，反而使總 token 消耗暴增超過原省下的成本。正確做法是縮減「輸入範圍」：精簡 CLAUDE.md 指令、限制 context 窗口大小、控制工具呼叫的參數深度。文件生成、測試撰寫等低複雜度任務可安全使用輕量模型，但一旦涉及「判斷」則應堅守頂級模型。此觀點挑戰了許多工程師的成本直覺，是部署 Claude Code 代理前必讀的思考框架。

*解決 Claude Code 跨 Session 性能退化問題的交接文件方案*
來源：Zenn - Claude Code｜發布：2026-07-03 03:47 JST

Claude Code 在長時間使用後因 context 積累會出現「性能退化」現象，後段指令遵循度下降。作者開發的 `hikitsugi` 工具在每個 session 結束時透過 `PostToolUse` Hook 自動生成交接文件（`/tmp/hikitsugi_*.md`），記錄已完成工作、待解問題及關鍵決策背景。下個 session 啟動時載入此文件，可顯著維持工作連續性，減少「重新解釋背景」的時間損耗。對超過 2 小時的持續開發任務效果最為明顯。此方案是應對 Claude Code session 隔離特性的實用解法，但交接文件本身的品質仍依賴上個 session 的最終狀態，需避免在代理出現混亂時結束 session。
