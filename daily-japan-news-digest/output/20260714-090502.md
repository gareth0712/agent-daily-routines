### [Claude Codeに「記憶」を持たせる — CLAUDE.md肥大化から抜け出す二層設計](https://zenn.dev/map_universe/articles/agent-memory-two-layer-design)
*為 Claude Code 賦予「記憶」——擺脫 CLAUDE.md 膨脹的二層架構設計*
來源：Zenn - Claude Code｜發布：2026-07-14 08:00 JST

作者提出 AI 程式設計 agent 的二層記憶架構，核心洞見是「賦予 agent 記憶」意味著策略性地將資訊注入未來 session 的 context window，而非訓練模型本身。**第一層（常駐記憶）**：CLAUDE.md 維持在 100 行以內，僅存規則與邊界，每次 session 自動注入。**第二層（可搜尋記憶）**：append-only JSONL 日誌，儲存決策、教訓與摘要。為防止資訊腐化，採用「supersede」模式——新決策以引用方式失效舊決策，而非覆寫，保留完整決策歷史。SessionStart hook 僅注入摘要與指標，避免直接注入全部歷史造成的 CLAUDE.md 膨脹問題。此框架源自作者數個月在生產環境執行多個 CLI agent 的實戰經驗。
