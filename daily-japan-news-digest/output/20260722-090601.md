### [Claude Codeの並行開発は git worktree に寄せる——複数クローンとDevContainerを捨てた理由](https://zenn.dev/azuma71/articles/mp-zenn-0016-20260720)
*Claude Code 並行開發應靠攏 git worktree——放棄多重 Clone 與 DevContainer 的理由*
來源：Zenn - Claude Code｜發布：2026-07-21 23:09 JST

作者評估三種 Claude Code 多工作階段並行方式後，強力推薦 `git worktree` 作為單一 repo 下管理多個 Claude Code 工作階段的最佳解——因為 Claude Code 直接編輯工作目錄檔案，多工作階段若共用同一目錄將導致衝突。具體指令為 `git worktree add ../myapp-feature -b feat/new-endpoint`，每個 worktree 可在獨立目錄運作並共享 `.git` 歷史記錄，相較多重 Clone（磁碟佔用大）或 DevContainer（資源消耗高）更為輕量。維護注意：刪除時務必使用 `git worktree remove`（非 `rm -rf`），並定期執行 `git worktree prune` 清理元資料；同一分支無法跨多個 worktree 同時 checkout。僅在需要環境隔離（如不同 Node.js 版本）時才升級至 DevContainer，作者建議先以 `git worktree add ../$(basename $PWD)-tmp -b tmp/try` 驗證工作流程後再正式採用。
