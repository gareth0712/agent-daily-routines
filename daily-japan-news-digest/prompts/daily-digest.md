# Daily Japan News Digest — 執行指令

**Model**: Sonnet (整個 routine 統一)
**預期執行時長**: 5–10 分鐘
**預期輸出**: 1 個 MD 檔 + 2 個 commit + 1 封 email

---

## 前置變數

執行時用以下指令取得：
```bash
DATE_JST=$(TZ=Asia/Tokyo date +%Y-%m-%d)          # 2026-04-22
TIMESTAMP_JST=$(TZ=Asia/Tokyo date +%Y%m%d-%H%M%S)  # 20260422-070015
ISO_JST=$(TZ=Asia/Tokyo date -Iseconds)            # 2026-04-22T07:00:15+09:00
```

收件者信箱從環境變數 `MY_EMAIL` 讀取。**絕對禁止**將此值寫入任何檔案或 commit message。

---

## 步驟 1：初始化 Log

Append 到 `logs/actions.log`：
```json
{"ts":"<ISO_JST>","level":"INFO","action":"routine_start","detail":{"date":"<DATE_JST>","timestamp":"<TIMESTAMP_JST>"}}
```

## 步驟 2：讀取 sources.json

用 Read 工具載入 `sources.json`。解析 `categories` 欄位。

## 步驟 3：抓取 RSS（對每個 category 的每個 source）

**迴圈**：遍歷 7 個分類、共 22 個 sources。

每個 source：
1. `WebFetch(source.url)` 取 RSS XML
2. 解析 XML，取 `<item>` 清單
3. 過濾 `<pubDate>` 在過去 `meta.lookbackHours` (24) 小時內
4. 每 source 最多取 `meta.maxItemsPerSource` (5) 條

**成功** → `actions.log`:
```json
{"ts":"<ISO>","level":"INFO","action":"fetch_rss","detail":{"category":"<key>","source":"<name>","items":<n>,"status":200}}
```

**失敗**（timeout / 404 / 解析錯誤）→ `errors.log`:
```json
{"ts":"<ISO>","level":"ERROR","action":"fetch_rss","detail":{"category":"<key>","source":"<name>","url":"<url>","error":"<message>"}}
```
→ **繼續下一個 source，絕不中止整體流程**。

## 步驟 4：過濾與摘要（對每個分類）

### 4a. 每分類總量控制
合併該分類所有 source 的文章後，按時間排序，取前 `meta.maxTotalItemsPerCategory` (15) 條。

### 4b. 抓正文 + 摘要

對每篇：
1. `WebFetch(item.link)` 取正文
2. 依 `category.summaryDepth` 產生繁體中文摘要：

| summaryDepth | 每條摘要 | 特殊處理 |
|---|---|---|
| `detailed` | 4–6 句 | 必列：實作要點、版本號、影響範圍；破壞性變更用 ⚠️ |
| `standard` | 2–3 句 | 標準摘要 |
| `brief` | 1–2 句 | 只留關鍵事實 |

### 4c. `claude_code` 分類的特殊要求

- 提到新功能、新 flag、新指令 → **必須**列具體名稱（例：`/loop`、`--add-dir`、`routines`）
- 提到版本號 → **必須**保留（例：v2.1.72）
- 提到破壞性變更 → **必須**用 ⚠️ emoji 前綴
- 提到競爭產品（Cursor、Copilot、Aider 等）→ 直接寫出，不迴避

### 4d. 語言規範
- 繁體中文
- 人名、公司名、產品名、日文專有名詞 → 保留原文
- 標題：保留原文（日文/英文），底下用 *斜體* 附中文意譯

記錄：
```json
{"ts":"<ISO>","level":"INFO","action":"summarize_category","detail":{"category":"<key>","items_summarized":<n>}}
```

## 步驟 5：產出 MD → `output/<TIMESTAMP_JST>.md`

### 5a. 分類顯示順序

依 `meta.priorityCategories` + `category.priority` 欄位排序：

1. 🤖 Claude Code 新聞（high）
2. 🇺🇸 美國 AI 大廠（high）
3. 🧠 日本 AI / ML 研究（medium）
4. 👨‍💻 日本工程社群（medium）
5. 🖥️ 日本科技業（medium）
6. ₿ 日本 Crypto（low）

### 5c. 寫檔

```bash
# Write 工具寫入
output/<TIMESTAMP_JST>.md
```

記錄：
```json
{"ts":"<ISO>","level":"INFO","action":"write_output","detail":{"path":"output/<TIMESTAMP_JST>.md","items_total":<N>}}
```

## 步驟 6：Git Commit #1（Output）

```bash
cd daily-japan-news-digest
git add output/
git commit -m "docs(japan-news): daily digest <DATE_JST>"
```

取得 short SHA：
```bash
SHA_1=$(git rev-parse --short HEAD)
```

記錄：
```json
{"ts":"<ISO>","level":"INFO","action":"git_commit","detail":{"step":"output","sha":"<SHA_1>"}}
```

## 步驟 7：寄 Gmail

### 7a. 讀 MD 轉 HTML

將 `output/<TIMESTAMP_JST>.md` 轉為 HTML email body，排版要求：

- **Claude Code 區塊**：放最頂，背景色 `#EBF5FF`，加 `<h2 style="color:#0066CC">` 凸顯
- 其他分類用 `<section>` 包，分類間加 `<hr style="border: 0; border-top: 1px solid #E5E7EB;">`
- 外部連結一律加 `target="_blank" rel="noopener noreferrer"`
- 手機友好：`<div style="max-width: 600px; font-size: 16px; line-height: 1.6;">`
- 字型：`font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", system-ui, sans-serif;`

### 7b. 寄送

用 **Gmail connector**：
- **to**: `$MY_EMAIL`（直接引用環境變數，勿寫死）
- **subject**: `📰 Daily Digest <DATE_JST>`
- **body** (HTML): 上述轉換結果

**成功**：
```json
{"ts":"<ISO>","level":"INFO","action":"send_email","detail":{"recipient":"<redacted>","status":"sent","subject_date":"<DATE_JST>"}}
```

⚠️ `recipient` 欄位**一律寫 `<redacted>`**，不可寫真實 email。

**失敗**（quota / connector error）：
```json
{"ts":"<ISO>","level":"ERROR","action":"send_email","detail":{"error":"<message>","subject_date":"<DATE_JST>"}}
```
→ 繼續下一步，MD 檔已存，可事後補寄。

## 步驟 8：結束 Log

計算執行時間（routine_start 到現在秒數）：
```json
{"ts":"<ISO>","level":"INFO","action":"routine_end","detail":{"duration_seconds":<n>,"status":"success","items_total":<N>,"errors_count":<errors.log 行數>}}
```

若 errors.log 有任何 ERROR 行，`status` 改為 `"partial_success"`。

## 步驟 9：Git Commit #2（Logs）

```bash
git add logs/
git commit -m "chore(japan-news): logs for <DATE_JST>"
SHA_2=$(git rev-parse --short HEAD)
```

**注意**：這筆 commit 的 log 記錄不寫入 actions.log（避免循環），由 step 10 的 push 記錄涵蓋。

## 步驟 10：Git Push

```bash
cd ..  # 回到 repo root
git push origin main
```

**push 成功**：routine 正常結束。
**push 失敗**：寫 errors.log（但這筆不會被 commit，會在下次執行時一起帶上），exit 1 讓 Routine 標記失敗。

---

## 失敗處理總表

| 失敗點 | 處理方式 | 是否中止 |
|---|---|---|
| 單一 RSS fetch 失敗 | 記 errors.log，跳過 | ❌ 繼續 |
| 某篇文章正文 fetch 失敗 | 記 errors.log，該篇跳過 | ❌ 繼續 |
| 整個分類無文章 | MD 顯示「本日無更新」 | ❌ 繼續 |
| 寫 output MD 失敗 | 記 errors.log，exit 1 | ✅ 中止 |
| Gmail 寄送失敗 | 記 errors.log | ❌ 繼續（有 MD 可事後補寄）|
| Git commit 失敗 | 記 errors.log，exit 1 | ✅ 中止 |
| Git push 失敗 | 記 errors.log，exit 1 | ✅ 中止 |

---

## 安全檢查清單（每次執行都驗證）

執行結束前，Claude **必須**自我檢查：

- [ ] `output/<TIMESTAMP>.md` 無任何 email 地址（`grep @` 應 0 行）
- [ ] `logs/actions.log` 無真實 email（`recipient` 欄一律 `<redacted>`）
- [ ] `logs/errors.log` 無真實 email
- [ ] Commit messages 無 email / secrets
- [ ] `sources.json` 保持原樣，未被修改

若任一項檢查失敗 → 寫 errors.log 記錄，但**不回滾已 commit 的內容**（避免殘留半成品）。下次手動處理。
